# UI 클론 5단계 계획: 나머지 필수 화면 및 디테일 완전 이식

**목표**: 대다모의 주요 페이지(메인, 커뮤니티, 게시글, 마이페이지, 기록)는 100% 긁어오기를 완료했습니다. 이제 사이트 구동에 필요한 나머지 2개 핵심 페이지(글쓰기, 통합검색)를 마저 100% 똑같이 구현합니다.

## 1. 검색 페이지 (`search.html`) 100% 이식
- **방법**: `https://daedamo.com/search` 의 원본 소스코드를 그대로 다운로드하여 `search.html` 생성.
- **수정사항**: 마찬가지로 "탈모" -> "다이어트", "모발이식" -> "위고비/마운자로" 등의 테마 키워드 치환 적용.
- **결과**: 사용자가 검색을 클릭했을 때 나타나는 검색 결과 레이아웃(게시물 탭, 썸네일 리스트, 페이징 등)을 대다모와 픽셀 단위로 똑같이 구성.

## 2. 게시글 작성 페이지 (`write.html`) 100% 이식
- **방법**: 대다모 글쓰기 페이지는 로그인이 필요해 외부에서 직접 긁어오기가 막혀있으므로, 대다모의 게시판 껍데기(`daedamo_story.html`)를 베이스로 삼고, 그 안에 대다모 전용 에디터 폼(제목 입력란, 위지윅 에디터 영역, 첨부파일 영역, 등록/취소 버튼)을 대다모 CSS 클래스 기반으로 직접 주입하여 **육안상 100% 똑같은 글쓰기 화면**을 만들어냄.
- **수정사항**: 키워드 치환 적용.
- **결과**: 게시글 쓰기 버튼을 누르면 대다모 특유의 에디터 폼 화면이 완벽하게 나타남.

## 3. 사이트 전역 GNB/사이드바 링크 활성화
- **설명**: 100% 긁어온 소스코드 내부에는 원본 대다모 링크(`href="https://daedamo.com/..."`)가 그대로 박혀있습니다.
- **작업**: 메인 메뉴, 게시판 탭, 글쓰기 버튼 등의 주요 링크를 우리 프로젝트의 `community.html`, `post.html`, `write.html` 등으로 연결되도록 스크립트로 일괄 패치.
- **결과**: 껍데기만 있는 상태를 넘어, **실제로 페이지 간 이동이 가능해져 진짜 사이트처럼 작동**하게 됨.

## 완료 조건
위 3가지 작업이 끝나면, 사용자는 메인 -> 게시판 -> 글 읽기 -> 글 쓰기 -> 검색 -> 마이페이지로 이어지는 **모든 웹사이트 경험을 대다모와 100% 똑같은 UI** 환경에서 즐길 수 있습니다.
