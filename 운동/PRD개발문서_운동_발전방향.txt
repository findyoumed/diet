================================================================================
WebGym(Web-based AI Gym) 기술 및 제작 고도화 회의 안건
================================================================================


이 안건은 서비스의 상업화 이전에 해결해야 할 기술적 부채와 제품의 완성도(Production-ready)를 높이기 위한 제작 관점의 논의 사항들을 담고 있습니다.


--------------------------------------------------------------------------------
안건 1. 핵심 AI 엔진 고도화 및 품질 검증 (Technology)
--------------------------------------------------------------------------------

현재 상황: noseLandmarker 기반의 상하 움직임(점프)만 감지하는 초기 단계임.

1-1. 분석 알고리즘 확장 (Squat, Push-up, Lunge)
    - 현재의 코(Nose) 기반 분석을 전신 스켈레톤(33개 포인트) 활용으로 확장 계획.
    - 각 운동 종목별 '정석 자세' 알고리즘(각도 기반) 설계 및 오차 범위 설정.
    - 기술적 한계: 배경 노이즈나 복장 인식 오류를 어떻게 극복할 것인가?

1-2. 장치별 성능 최적화 (Web Performance)
    - 모바일 등 저사양 기기에서 MediaPipe(WASM/GPU) 성능 체감 지수 확인.
    - 카메라 FPS 저하 시 점수 계산 보정 로직 필요 여부.
    - 초기 로딩 시간(Pose Landmarker 모델 로드) 단축 전략.

결정 필요 사항: 다음 단계에서 구현할 핵심 운동 종목 2종 선정.


--------------------------------------------------------------------------------
안건 2. 제작 워크플로우 및 유지보수성 강화 (Production)
--------------------------------------------------------------------------------

현재 상황: 바닐라 JS 기반으로 매니저 객체들을 수동 주입하는 구조.

2-1. 코드 구조 개선 (Refactoring & Architecture)
    - `script.js` 등 메인 파일의 복잡성 해결 (순환 참조 문제 해결).
    - '파일당 250줄 제한' 규칙을 준수하면서도 기능 확장이 용이한 플러그인 구조 논의.
    - 상태 관리(State Management) 방식 통일 (현재 LocalStorage와 변수 혼용).

2-2. 자동화 및 배포 프로세스
    - Vercel 배포 전 코드 린트(Lint) 및 문법 체크 자동화.
    - PWA(Progressive Web App) 오프라인 모드 및 캐시 갱신 전략 (sw.js 고도화).
    - 브라우저 호환성 테스트 (Safari, Chrome, Samsung Internet 등).

결정 필요 사항: CI/CD 도입 여부 및 코드 컨벤션 가이드라인 확립.


--------------------------------------------------------------------------------
안건 3. 사용자 경험(UI/UX) 고도화 전략 (User Experience)
--------------------------------------------------------------------------------

현재 상황: 기본 레이아웃 및 점수 표시 등의 핵심 UI만 구현된 상태.

3-1. 실시간 시각적 피드백 시스템
    - 자세 정확도에 따른 컬러 가이드 (초록: 정석, 빨강: 위험).
    - 콤보(Combo) 및 피버(Fever) 모드 진입 시 시각 효과 (Particle, Glow) 구현.
    - 운동 중 시선 방해를 최소화하는 반투명 오버레이 UI 최적화.

3-2. 모바일 퍼스트(Mobile-First) 디자인 강화
    - 가로/세로 화면 전환에 따른 완벽한 반응형 레이아웃 검증.
    - 운동 중 조작이 불필요한 음성 커맨드 또는 제스처 UI 검토.
    - 온보딩(Onboarding) 프로세스 개선: 첫 방문자가 10초 이내에 운동을 시작할 수 있게.

결정 필요 사항: 운동 중 가장 중요한 시각 지표(점수 vs 정확도 vs 시간) 우선순위 결정.


--------------------------------------------------------------------------------
안건 4. 데이터베이스 및 서버 아키텍처 (Database & Infrastructure)
--------------------------------------------------------------------------------

현재 상황: 대부분의 데이터를 클라이언트 사이드(LocalStorage)에 저장 중.

4-1. 백엔드 서비스 연동 (Supabase / Firebase)
    - 사용자 운동 기록(Log) 및 플레이리스트 저장용 DB 스키마 설계.
    - 실시간 리더보드(Ranking) 구현을 위한 데이터 처리 방식.
    - 소셜 로그인(Kakao, Google) 도입 시 사용자 정보 관리 전략.

4-2. 데이터 동기화 및 오프라인 처리
    - 기기 변경 시 운동 기록 동기화 기능.
    - 네트워크 불안정 시 로컬에 먼저 저장 후 업로드하는 '낙관적 업데이트' 적용.
    - 개인별 운동 성과 지표(대시보드) 분석에 필요한 필수 데이터 정의.

결정 필요 사항: 백엔드 솔루션 선정 (Supabase vs 자체 서버) 및 핵심 DB 테이블 정의.


--------------------------------------------------------------------------------
안건 5. 사용자 제작 콘텐츠(UGC) 및 연동 (YouTube Production)
--------------------------------------------------------------------------------

현재 상황: 유튜브 API를 통해 플레이리스트에서 영상 호출 중.

5-1. 유튜브 영상 동기화 기술 (Sync Tech)
    - 영상의 운동 구간(시작/종료 초 단위) 정보를 외부에서 동적으로 불러오는 기술.
    - 강사의 동작과 사용자의 동작을 시간차 없이 비교하는 동기화 로직 고도화.

5-2. 플레이리스트 제작 편의성
    - 사용자가 직접 유튜브 링크를 넣어 '자신만의 루틴'을 만들 때의 UX.
    - 영상별 운동 난이도 및 종목(스쿼트 등)을 태깅하는 시스템 구축.

결정 필요 사항: 운동 구간 메타데이터(JSON/DB) 관리 방식 결정.


--------------------------------------------------------------------------------
안건 6. 상업화 도입을 위한 기술적 장치 (Pre-Commercialization)
--------------------------------------------------------------------------------

6-1. 데이터 보안 및 프라이버시 검증
    - "로컬 처리"를 입증할 수 있는 기술적 증명 (네트워크 탭 검증 등).
    - 사용자가 공유를 위해 생성하는 이미지의 보안성 (캔버스 API 처리 방식).

6-2. 유료화 기능의 기술적 구현 (Feature Flags)
    - 로그인/DB 없이 구독 여부를 확인할 수 있는 방안 논의 (예: 토큰 기반).
    - 프리미엄 기능(고퀄리티 리포트 생성)을 위한 서버 사이드 연동 계획.

결정 필요 사항: 로그인 없는 회원 식별이 가능한지(Fingerprinting 등) 기술 검토.


--------------------------------------------------------------------------------
안건 7. 테스트 및 QA 전략 (Quality Assurance)
--------------------------------------------------------------------------------

7-1. AI 자세 인식 실효성 테스트
    - 다양한 체형, 배경, 조명 환경에서의 인식률 테스트 시나리오 수립.
    - '부정 행위(치팅)' 방지 로직 (카메라 고정 확인 등).

7-2. 사용자 피드백 루프 (PMF 분석 고도화)
    - 현재의 PMF 모달 외에 기술적 오류(AI 인식 불가 등) 자동 리포팅 시스템.

결정 필요 사항: 베타 테스터 선정 및 피드백 수집 기간 확정.

핵심 마일스톤 (Technical Roadmap):
- 1단계: 멀티 포인트 알고리즘 도입 (스쿼트용) 및 DB 스키마 설계
- 2단계: 코드 아키텍처 재정의 및 UI 시각 피드백(콤보 효과) 강화
- 3단계: 백엔드 연동(Supabase) 및 유료화 대비 보안 기술 검토
