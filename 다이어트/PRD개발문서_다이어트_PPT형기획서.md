# [슬라이드 2] 서비스 기획 배경 및 가설

## 🔍 기존 서비스의 문제점과 우리의 해결책

* **문제 정의**
  - 기존 앱(지니어트 등)은 입력 지표가 너무 많아 유저 피로도가 높음
  - 게시판의 깊이가 낮아 단순 친목 수준에 머무름
* **핵심 가설**
  - "체중/식단 기록에 포인트를 지급하고, 복약 관리를 제공하면 유저 리텐션이 유지될 것이다."
* **차별화 포지셔닝**
  - 기록 최소화 (체중, 식단 2가지만)
  - 비만치료제(위고비 등) 복약 추이 차트 제공
  - G포인트 리워드 제공으로 락인(Lock-in) 효과 극대화

# [슬라이드 3] MVP 기능 범위 정의

## ❌ 이번 버전에서 제외할 기능 (Out of Scope)
* **복잡한 건강 지표**: 혈당, 혈압, 물 섭취량, 배변 상태, 컨디션, 눈바디 사진
* **소셜 로그인 다변화**: 네이버, 구글, 애플 등 (카카오 1개만 지원)
* **게시판 세분화**: 카테고리 분리 배제 (자유게시판 1개 통합)
* **포인트 교환 상점**: 커머스 및 기프티콘 교환 기능 (추후 오픈)

## ✅ 핵심 구현 기능 (In Scope)
* **인증**: 카카오 소셜 로그인 연동
* **기록**: 일일 체중 입력 & 식단 칼로리 검색 다이어리
* **소셜**: 태그 필터링 기반 1개 자유게시판
* **리워드**: 체중/식단 입력 시 실시간 포인트 지급 및 적립 내역 조회

# [슬라이드 4] 카카오 로그인 및 회원 설정

### 🔑 요구사항 및 데이터 구조

* **카카오 로그인 SDK 연동**
  - 사용자가 로그인 버튼 클릭 시 카카오 인가 코드를 수신
  - Supabase Auth 세션과 연동하여 회원 정보 테이블 자동 등록
* **최초 정보 입력 폼**
  - 닉네임 (한글/영문 2~10자)
  - 신장 (100cm ~ 250cm 제약 조건)
  - 목표 체중 (20kg ~ 300kg 제약 조건)
* **실시간 BMI 계산기**
  - 공식: `BMI = 체중(kg) / (신장(m))^2`
  - 체중 입력 시 화면 상단 프로필에 BMI 지수 실시간 연산 노출

# [슬라이드 5] 체중 및 복약 기록 (Weight Tracker)

### ⚖️ 일일 기록 및 꺾은선 차트 시각화

* **체중 기록 규칙**
  - 당일 데이터는 제한 없이 입력 및 수정 가능 (달력 네비게이션 제공)
  - 소수점 첫째 자리까지 입력 지원 (`decimal(4,1)`)
* **복약 체크 규칙**
  - 오늘 비만치료제(위고비 등) 또는 다이어트 보조제 투약 여부 토글스위치 선택
* **30일 체중-복약 매칭 그래프**
  - 최근 30일간의 몸무게 변화를 꺾은선 차트(Chart.js CDN)로 렌더링
  - 투약일(`medicine_taken = true`) 데이터 포인트에는 주사기 모양 아이콘 표시

# [슬라이드 6] 식단 칼로리 다이어리 (Diet Diary)

### 🥗 음식 DB 검색 및 영양소 구성 분석

* **한국인 대표 다이어트 식품 DB 탑재**
  - 식약처 데이터를 정제한 3,000개 로컬 JSON 파일 빌드
  - 한글 초성 검색 및 입력값 디바운싱(200ms) 처리로 속도 보장
* **식단 입력 상세**
  - 아침, 점심, 저녁, 간식 별로 음식을 추가하고 섭취량(g) 입력
  - 오늘 섭취한 총 열량 칼로리 실시간 합산
* **탄단지 원형 도넛 차트**
  - 섭취한 영양소의 그람(g) 수를 열량으로 환산하여 비율 계산
  - `Chart.js`를 사용해 탄/단/지 구성 비율을 도넛 차트로 실시간 드로잉

# [슬라이드 7] 태그형 자유게시판 (Community)

### 💬 단일 게시판 집중 및 필터링 검색

* **단일 자유게시판 운영**
  - 모든 작성 글은 "다이어트 이야기" 단 하나의 타임라인에 시간순 정렬
* **태그 지정 필수 정책**
  - 글 작성 시 `[식단인증]`, `[치료제후기]`, `[부작용Q&A]`, `[자유수다]` 필수 지정
* **필터 바 UI 제공**
  - 상단 필터 단추 클릭 시 선택된 태그의 게시글만 화면에 실시간 검색 노출
* **미디어 업로드 제한**
  - 게시글당 이미지는 최대 3장까지만 첨부 허용
  - 파일 용량 제한: 장당 최대 3MB (Supabase Storage 정책 적용)

# [슬라이드 8] G포인트 적립 정책 (Reward System)

### 💰 보상 규칙 및 어뷰징 차단 설계

| 행동 구분 | 지급 포인트 | 일일 한도 기준 |
|---|---|---|
| 회원 가입 축하 | **1,000P** | 최초 1회 지급 |
| 일일 체중 입력 | **50P** | 하루 최대 1회 |
| 일일 식단 기록 | 끼니당 **50P** | 하루 최대 3끼 (최대 150P, 간식 제외) |

* **어뷰징 방지 제약조건**
  - DB 수준에서 유저 ID, 날짜, 적립 행동의 고유키 조합 제약 설정
  - 중복 적립 호출 시 DB 무결성 제약조건(`user_id + record_date + action`)으로 자동 차단
* **G포인트 화면**
  - 마이페이지 내에서 일자별 적립 사유와 포인트를 테이블 형식으로 제공

# [슬라이드 10] 개발 로드맵 및 성공 지표

### 📅 5주 완성 타임라인 & Go/No-Go 기준

* **개발 일정**
  - **1주차**: 개발 환경 구축, Supabase 연동 및 카카오 로그인 완료
  - **2주차**: 로컬 식약처 3,000개 음식 데이터 검색 및 디바운싱 처리 완성
  - **3주차**: 체중/식단 기록 UI 완성 및 LocalStorage 포인트 적립 연동
  - **4주차**: 태그형 게시판 CRUD 및 좋아요/댓글 기능 빌드
  - **5주차**: 마이페이지 포인트 히스토리 연동, 종합 QA 및 배포 완료
* **Go/No-Go 판단 기준 (출시 4주차 검증)**
  - 가입자 수 200명 이상 달성
  - **14일 기록 리텐션(유지율) 30% 이상** 달성 시 Phase 2 고도화 결정
