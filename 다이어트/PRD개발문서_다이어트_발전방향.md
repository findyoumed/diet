================================================================================
다이어트온(DietOn) 기술 및 제작 고도화 회의 안건
================================================================================


이 안건은 서비스를 시장에 출시하기 전 해결해야 할 기술 부채와 제품의 완성도를 높이기 위해 논의할 제작 관점의 주제들입니다.


--------------------------------------------------------------------------------
안건 1. 핵심 칼로리 계산기 및 음식 DB 정밀화 (Technology)
--------------------------------------------------------------------------------

현재 상황: 식약처 데이터 중 대표적인 3,000개만 선별해 사용자 브라우저(로컬)의 JSON 파일로 실어놓은 상태입니다.

1-1. 검색 알고리즘 속도 및 한글 초성 검색 구현
    - 사용자가 한글 초성을 입력할 때 실시간으로 디바운싱(대기 처리)을 적용해 검색 응답 속도를 끌어올립니다.
    - 기술적 제약: 모바일 기기의 브라우저에서 3,000여 개의 데이터를 정규식으로 계속 검색할 때 발생할 수 있는 렌더링 지연 및 연산 부하 해결.

1-2. 외부 API 도입 연동
    - 로컬 데이터 외에 바코드로 편하게 음식을 스캔할 수 있도록 외부 식품 API와 연동하는 방안 검토.
    - 식품 영양성분 정보가 바뀔 때를 대비한 데이터 갱신 주기 관리.

결정할 사항: 한글 초성 검색 알고리즘 라이브러리를 직접 작성할지, 기존 도구를 채택할지 결정.


--------------------------------------------------------------------------------
안건 2. 제작 워크플로우 및 코드 구조 개선 (Production)
--------------------------------------------------------------------------------

현재 상황: 프레임워크 없는 순수 자바스크립트로 구현하며 각 매니저 객체(저장소, UI, 포인트)를 수동으로 연결해 사용하고 있습니다.

2-1. 코드 구조 개선 (Refactoring & Architecture)
    - app.js 같은 핵심 스크립트 파일에 코드와 로직이 몰려 복잡해지는 현상을 해결하고, 순환 참조 오류를 예방합니다.
    - 파일 하나당 250줄을 넘기지 않으면서도 추후 기능을 덧붙이기 쉬운 유연한 구조를 세웁니다.
    - 상태 관리의 일원화 (현재 로컬스토리지와 임시 전역 변수가 혼재된 상태를 정리).

2-2. 자동화 및 배포 프로세스
    - 코드를 배포하기 전 구문 에러나 가독성을 잡아내기 위한 코드 린트(Lint) 자동화 도입.
    - 다양한 모바일 브라우저(아이폰 사파리, 삼성 인터넷 등)에서 레이아웃이 깨지지 않도록 하는 최적화 테스트.

결정할 사항: 코딩 스타일 가이드라인 확립 및 린트 도구 선정.


--------------------------------------------------------------------------------
안건 3. 데이터베이스 및 서버 아키텍처 (Database & Infrastructure)
--------------------------------------------------------------------------------

현재 상황: 대부분의 건강 일지와 포인트 정보를 사용자 기기의 로컬스토리지에만 저장하고 있습니다.

3-1. Supabase 연동 및 동기화 기술 (Sync Tech)
    - 로그인하는 순간 로컬에 쌓아둔 포인트 적립 내역을 서버의 데이터베이스로 안전하게 복사 및 갱신해주는 동기화 프로세스 설계.
    - 인터넷이 불안정할 때 데이터가 유실되지 않도록 보장하는 오프라인 우선(Offline-First) 처리 고도화.

결정할 사항: 데이터 무단 접근 방지를 위한 Supabase RLS 규칙 설정 및 동기화 주기 확정.


--------------------------------------------------------------------------------
안건 4. 상업화 도입을 위한 헬시딜 커머스 연동 (Pre-Commercialization)
--------------------------------------------------------------------------------

4-1. 포인트 결제 연동 기술
    - G포인트를 사용해 쇼핑몰에서 결제할 때 1P = 1원 가치로 안전하게 차감 결제하는 연동 방안.
    - 클라이언트에서 포인트를 조작하는 비정상 어뷰징을 막기 위해, 통신 암호화 및 유효성 검증 서버 로직 추가.

결정할 사항: 포인트 결제 제휴 모델 및 환전 정책 확정.
